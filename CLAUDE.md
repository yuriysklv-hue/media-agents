# media-agents — контекст для Claude

Пайплайн ИИ-агентов, наполняющий медиа **1screen.ru** новостями об adtech и маркетинге. Собирает англоязычные RSS, фильтрует, переводит и переписывает на русский, прогоняет через QA и **создаёт Pull Request в репозиторий `yuriysklv-hue/media`** (`media-site/src/content/news/`). На сайт материалы попадают только после ручного merge PR.

> Общий контекст проекта (бренд, позиционирование, дизайн сайта, деплой на Timeweb) — в `CLAUDE.md` репозитория **`media`**. Здесь — только про пайплайн. В новой сессии репозитории подключаются командой «добавь репозиторий media-agents» / «…media» (`add_repo`).

### Сессия 10.07.2026 (PR смёржен): фикс `pubDate`

- **`pubDate` = дата выхода на 1screen (UTC now), а не дата источника.** Раньше `news_writer` ставил `pubDate = event["published_at"]` (дата в RSS) → статьи датировались задним числом, выглядели несвежими, не зажигали live-точку (`isLive` <3 ч) и тонули внизу ленты (сортировка по `pubDate` убыв.). Теперь детерминированная доводка front-matter вынесена в `news_writer._finalize_meta`, где `pubDate = utcnow_iso()`.
- **Дата источника не потеряна** — паблишер пишет её в `published.jsonl` отдельным полем `source_published_at` (`git_publisher._record_published`). Во front-matter она **не идёт** — чтобы не расширять контракт схемы сайта (риск падения сборки Astro на строгом `source`).
- **Только forward-looking:** правка не трогает уже опубликованные `.md` в репо `media` (их `pubDate` остаётся как есть), дедуп по `seen_urls` исключает перегенерацию старых материалов. Эффект — на следующей партии.
- Промпт `news_writer.md`: `pubDate: "AUTO"` (значение перетирается кодом). Тесты: `tests/test_content_feedback.py` (`test_pubdate_*`, `test_finalize_sets_source_and_defaults`).

## Как устроен пайплайн

`run_pipeline.py` → этапы (`src/pipeline.py`): **collect → pre_filter → translator → filter_dedup → write → publish**

- **collect** (`src/collectors/rss_collector.py`) — RSS-фиды (Adweek, AdExchanger, Digiday, MediaPost…), ~100 сырых материалов.
- **pre_filter** (`src/processors/pre_filter.py`) — keyword-скоринг + GLM-скор релевантности.
- **translator** (`src/processors/translator.py`) — DeepSeek EN→RU (+ кэш переводов в `data/state/`).
- **filter_dedup** (`src/processors/filter_dedup.py`) — дедуп. Семантический (эмбеддинги) **отключён на z.ai** (ветка дремлет, оживёт при смене провайдера); работают URL-дедуп + **TF-IDF по символьным n-граммам** (`_tfidf_events`, `src/utils/tfidf_engine.py`, scikit-learn `char_wb` 3–5): отсев дублей опубликованного (по заголовку) и склейка одной новости из разных фидов в событие (по заголовок+summary, мультиисточник → `additional_sources`). Пороги эмпирические, env-override `DEDUP_DUP_THRESHOLD`/`DEDUP_CLUSTER_THRESHOLD` (17.07.2026, Фаза 1 дедупа). Прежний текстовый фолбэк (`text_similarity.py`) больше не в контуре, модуль оставлен.
- **write** — DeepSeek пишет статью (`src/writers/news_writer.py`, после генерации `legal.add_restricted_org_footnotes` ставит сноску Meta/FB/Instagram); enricher (GLM) проставляет slug/description/category/geo/tags/author/social_title (`src/processors/enricher.py`, description укладывается в 160 законченной фразой); QA (GLM + детерминированные проверки, `src/processors/qa.py`) бракует слабые тексты в `data/drafts/failed/`.
- **publish** (`src/publishers/`) — коммит прошедших QA материалов в ветку репо `media` + создание PR.

## LLM-клиент — важные детали

`src/llm_client.py` — единый OpenAI-совместимый клиент. Провайдеры/модели в `config/models.yaml`, переопределяются env: `ZHIPU_BASE_URL`, `GLM_FLASH_MODEL`, `DEEPSEEK_BASE_URL` и т.п. (env приоритетнее config).

- **GLM** = `glm-4.5-flash` на **z.ai** (`https://api.z.ai/api/paas/v4`) — служебные JSON-этапы (pre_filter, enricher, qa, filter_dedup). **DeepSeek** = `deepseek-chat` — перевод и написание.
- ⚠️ **GLM thinking отключён принудительно.** `glm-4.5/4.6` на z.ai по умолчанию включают reasoning → тратят `max_tokens` на рассуждение и возвращают **пустой `content`**, из-за чего JSON-этапы падали на `json.loads("")` и уходили в фолбэк, а прогон упирался в тайм-аут. Клиент шлёт `extra_body={"thinking":{"type":"disabled"}}` для GLM-вызовов. **Не включать обратно** (аварийный откат — секрет `GLM_DISABLE_THINKING=0`).
- **Эмбеддинги z.ai не поддерживает** — вызов падает с 400, семантический дедуп деградирует до URL-дедупа (заложено, не баг).
- ⚠️ **Локально LLM протестировать нельзя** — ключи только в GitHub Secrets. Правки в LLM-логике проверяются прогоном воркфлоу (Actions → Run workflow), лог читается через MCP.

## Контракт с сайтом (репо `media`)

Схема материала — `media-site/src/content.config.ts`. Пайплайн обязан соблюдать (иначе сборка Astro на Timeweb падает и сайт не обновляется):

## Открытые задачи и грабли

Актуальный список — в **`NEXT_SESSION.md`**. Ключевое:

- **Промпты / качество (главное, задача 4a):** QA бракует часть текстов как «ИИ-голос». Гибкий промпт + хирургическая петля ревизии (`reviser.md`) уже смёржены и чинят **точечные** дефекты, но **диффузный** ИИ-голос ими не лечится. Главный следующий рычаг — **образцы (few-shot) в `src/prompts/news_writer.md`**: показывать модели удачные формулировки, а не только запрещать плохие. Эталоны курирует владелец (кандидаты — из репо `media`). Сверять с `src/prompts/qa_style.md`.
- **Корпус good/bad:** забракованные QA черновики (`data/drafts/failed/`) живут на эфемерном раннере и теряются — сохранять как artifact/коммит с причиной QA, чтобы был размеченный материал для тюнинга.
- **`pubDate` = дата выхода на 1screen (UTC now)** — исправлено 10.07.2026 (`news_writer._finalize_meta`). Раньше ставилась дата источника из RSS → материалы датировались задним числом, выглядели несвежими и тонули в ленте. Дата источника теперь сохраняется отдельно в `published.jsonl` (`source_published_at`), во front-matter не идёт (контракт схемы сайта не расширяем).
- **Баги заголовков — закрыты 13.07.2026 (PR #18):** (1) двоеточие в заголовке ломало YAML front-matter (`mapping values are not allowed here`) → весь материал терялся — теперь `split_front_matter` на `yaml.YAMLError` чинит незакавыченные строковые поля (`_repair_front_matter` в `src/utils/frontmatter.py`) и парсит повторно; (2) заголовки вне 50–80 бракуются QA — теперь `news_writer.write_news` делает один авто-ретрай с корректирующим хинтом при промахе длины. Тесты: `tests/test_content_feedback.py`.
- **Дедуп — Фаза 1 СМЁРЖЕНА и проверена боем (17.07.2026, PR #22 в `media-agents` + PR #50 в `media`):** TF-IDF (символьные n-граммы) заменил текстовый фолбэк — плотнее, ловит перефразировки одного сюжета; слияние источников в батче → видимый `additional_sources` (детерминированно, QA валидирует, иначе сборка Astro падает); `failed_drafts.jsonl` (персист брака); тонкий `ArticleStore` (шов под PostgreSQL); кросс-репо схема сайта (`additional_sources`/`updatedDate` в `content.config.ts` + рендер + JSON-LD `dateModified`). **Run #28 (тест-режим):** 3 корректные склейки, включая перефразировки (Netflix 0.86, CIMM 0.87), ложных склеек нет, QA 11/11, $0.03. Пороги (`DEDUP_DUP_THRESHOLD=0.62`/`DEDUP_CLUSTER_THRESHOLD=0.45`) серую зону 0.45–0.85 ещё не задели — оставлены, докрутка env по боевым прогонам. **Осталось:** update-flow (междневное тихое обновление статьи) → **Фаза 2** (бэклог в `NEXT_SESSION.md`).
- **Тест пайплайна с фиче-ветки — только через тест-режим.** С тех пор как персист состояния заработал (задача 7, PR #16), фиче-ветка наследует `seen_urls` из `main`, а cron на `main` уже собрал те же фиды → обычный прогон воркфлоу с ветки собирает **~0 материалов** (`пропущено N как уже виденные`), этап write голодает. Плюс шаг «Commit data state» на ветке краснит (add/add rebase). **Решение — тест-режим (реализован):** `News Pipeline` → Run workflow → галочка **`test_mode`**. Что делает (env `PIPELINE_TEST_MODE`): (1) dry-run — без PR в `media` и без коммита состояния (шаг «Commit data state» пропускается → джоба зелёная); (2) обход дедупа — коллектор игнорирует `seen_urls` и НЕ сохраняет его (боевую память не трогает); (3) защита от архивного хвоста — окно свежести `TEST_MAX_AGE_DAYS=3` на ВСЕ фиды + существующий кап 120/фид; (4) твёрдый потолок объёма — в перевод/написание уходит топ-`TEST_MAX_ITEMS=15` самых релевантных (гарантия по стоимости и 45-мин лимиту); (5) черновики (прошедшие + брак) выгружаются артефактом `drafts-<run_id>`. Числа переопределяются env-переменными. Реализация: `rss_collector.collect_rss`, `pre_filter.run_pre_filter`, `run_pipeline.main`, `news-pipeline.yml`; тесты `tests/test_test_mode.py`. ⚠️ НЕ чистить `seen_urls`/`published.jsonl` коммитом в ветку ради теста — при merge в `main` затрёт боевую память дедупа (переопубликует всё); тест-режим именно поэтому ничего в состоянии не персистит.
