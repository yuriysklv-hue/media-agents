## ⚠️ Задача 1 (БЛОКЕР автономности): влить фикс GLM в `main`

Плановые прогоны идут по cron **с `main`** (09:00/21:00 МСК). Пока фикс не в `main`, они запускают **старый сломанный код** (GLM-thinking баг + 30-мин тайм-аут) и будут падать. **Смёржить PR `claude/media-agents-setup-svyuez` → `main`** — первым делом. После мёржа заодно уходит косметический баг с «Commit data state» (см. задачу 7).

## Задача 2: механизм корпуса «качественные / некачественные»

Владельцу нужно докручивать агента написания на реальных примерах. Сейчас:
- **хороший корпус** = опубликованные статьи (в репо `media`, `media-site/src/content/news/`);
- **плохой корпус** = забракованные QA черновики уходят в `drafts/failed/` **на эфемерном раннере и теряются** — их не видно.

Надо **сохранять забракованные черновики + вердикт QA (с причиной)** между прогонами: либо коммитить `data/drafts/failed/` в репо, либо загружать как artifact воркфлоу (`actions/upload-artifact`). Тогда у владельца будет размеченный набор «прошло / не прошло + почему» для тюнинга промптов. Причины QA уже пишутся в лог — можно дополнительно собирать их в сводный отчёт (slug → вердикт → причина).

## Задача 3 (отдельная, для владельца): промпты и качество текстов

Главный сигнал первого прогона: **7 из 11 отказов QA — это «ИИ-голос».** GLM-QA стабильно бракует тексты формулировкой вроде *«слишком гладкий, безликий и шаблонный, нет авторской позиции»*. Это центральная цель тюнинга `src/prompts/news_writer.md`: живее, авторская интонация, меньше предсказуемых подзаголовков/шаблонных абзацев. Параллельно — выверить, не слишком ли строг сам QA-порог (`src/prompts/qa_style.md` / пороги в `src/processors/qa.py`).

## Задача 4 (баг): двоеточие в заголовке ломает front-matter

Один материал упал: `mapping values are not allowed here` — DeepSeek поставил `:` в значение YAML без кавычек. Квотить строковые значения front-matter (`title`, `description`, `social_title`) в паблишере/райтере, либо жёстко требовать экранирование в промпте.

## Задача 5 (баг): длина заголовка вне 50-80

`news_writer` выдаёт заголовки 44/45/91 симв. → бракуются QA. Усилить ограничение в `src/prompts/news_writer.md` (явно: 50-80 символов, проверять) или добавить авто-ретрай при промахе.

## Задача 6: дедуп-дубли без эмбеддингов

Без семантического дедупа одна новость из двух RSS-фидов даёт 2 статьи (в этом прогоне поймали DuckDuckGo руками — обе ссылались на статью 416389 MediaPost, но пришли из фидов TV и Media). Варианты: URL-канонизация (нормализовать/обрезать до article-id), дедуп по заголовку (fuzzy), либо отдельный провайдер эмбеддингов (bigmodel-ключ или другой).

## Задача 7 (косметика): «Commit data state» падает с фиче-ветки

Финальный шаг воркфлоу делает `git pull --rebase origin main`; при запуске с фиче-ветки это конфликтует (add/add) и красит job красным уже **после** создания PR. На `main` шаг проходит чисто — после задачи 1 проблема уходит. Опционально: сделать шаг устойчивым (`|| true`, или коммитить состояние только на `main`).
